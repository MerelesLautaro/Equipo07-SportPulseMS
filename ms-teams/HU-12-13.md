## HU-13 · Detalle de un equipo

Como usuario autenticado (o como otro microservicio), quiero consultar la información completa de un equipo por su ID, para obtener sus datos de estadio y fundación.

Criterios de aceptación:

- GET /api/teams/{teamId} devuelve el detalle completo del equipo

- La respuesta incluye los datos del estadio: nombre, dirección, ciudad, capacidad y tipo de superficie

- Si el ID no existe en API-Football, responde con código 404 y el error TEAM_NOT_FOUND

- El endpoint es accesible tanto por usuarios autenticados como por llamadas internas de otros microservicios

## HU-12 · Listado de equipos por liga y temporada
Como usuario autenticado, quiero consultar los equipos que participan en una liga durante una temporada específica, para explorar la información de cada club.

### Criterios de aceptación:

- GET /api/teams requiere los parámetros league y season; si falta alguno, responde con código 400

- La respuesta incluye id, nombre, país, logo, año de fundación y estadio de cada equipo

- Los datos provienen de API-Football y se transforman al formato del enunciado

- El endpoint requiere token JWT válido
